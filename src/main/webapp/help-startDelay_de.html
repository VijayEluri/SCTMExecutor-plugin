Wenn sie mehr als eine Testsuite zur Ausführung configuriert haben, können sie hiermit eine kaskadierende Verzögerung der zweiten und aller folgenden Testsuiten in Sekunden spezifizieren.
Das verzögerte Ausführen kann helfen um die Last beim Starten (z.B.: einer virtuallen Maschine) einer Testsuite zu minimieren.
Notiz: Wenn eine parallele Ausführung nicht möglich ist wird empfohlen ein weiteres Buildverfahren (Build-Schritt) anzulegen.
