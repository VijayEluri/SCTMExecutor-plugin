Choose an option how the build number should be provided to Silk Central:

  • Don't care about a build number: No build number is used to start the test execution. It depends how Silk Central is configured, but typically the execution definition are configured with a build number, which is used for the execution. For more information read the documentation of Silk Central. This option should also be used if you write a 'buildinfo-file' by your own.
  • Use last build number added on Silk Central: Use the latest build number for this product and version configured on the Silk Central. For more information how you can configure products and versions read the documentation of Silk Central.
  • Use build number from this job: Use the build number from the current Jenkins job and create it in Silk Central if not exists.
  • Use build number from upstream job: For this option the build number of the selected upstream job is taken for the execution in Silk Central.
